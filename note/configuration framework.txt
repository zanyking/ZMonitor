

App的責任:
	app 才有環境知道configuration可以從哪來
		+. 在pure Java program裡才知道可以從sysprops、args來。
		+. 在Servlet裡才知道config可以從/WEB-INF/zmonitor.xml來。
		+. 在eclipse plugin裡才知道config 可以從plugin base, workspace來。
	環境決定URL
	環境決定要用哪種ConfigurationSource
		FileConfigurationSource

ConfigurationSource的責任：
	由環境決定要用哪一種
	讀取configuration(the xml file)
	偵測configuration是否有變化
	必要時reset ZMonitor


ConfiguratorLoder:
	啟動時，Loder 會讀取所有的 META-INF/zmonitor/config.properties，取得configurator class
		configurators=a.b.c, d.e.f, k.n.g
	PropertyConfigurator：根據config.properties裡其他的屬性資料，決定
		[fqcn, ]
		web-conf=org.zmonitor.web.JavaWebConfiguration
		url-filer=org.zmonitor.web.filter.DefaultUrlFilter
		condition=


ZMonitor-Core:
	MSHandler
	Internal Log

ZMonitor-Web:

	ZMonitor Init
	ZmonitorFilter(decide which request should be monitored)

ZMonitor-zk:
	ZMonitor Init

ZMonitor-Log4J, Logback, Slf4j:
	ZMonitor Passive Init


ZMonitor-Agent:
	Agent instance configuration(as a ZMonitor prop)
